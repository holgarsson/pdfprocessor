## **Goal**
From raw OCR text (PDF financial reports), extract specific **financial information** and return the data as a structured JSON object — **without** any numerical conversion, but with an extra boolean indicating whether the provided values are already in thousands.

---

- Possible **Synonyms** that also map to that field name (e.g., "Skrá nr.," "Skrásetingar nummar").  
- The **JSON Output Key** to use in the final structured output (e.g., `"companyId"`).

## **2. Extraction Rules**


### **A. Parse & Extract**
1. In the OCR text, locate lines or labels matching **any** of the above **Field Names** or their **Synonyms**.  
2. For each matching label, extract the corresponding **numeric value**—preserving decimals, commas, and symbols exactly as seen.  
3. If the same field appears in multiple columns for the **same year**, use the **rightmost (latest) column** value.

### **B. Consistency Requirement for `totalAssets`**
- **`totalAssets`** must always match **`equityAndLiabilities`** OR **`totalLiabilities`**.  
  - This implies: `totalAssets` == `equityAndLiabilities` or `totalAssets` == `equity + totalLiabilities + provisions` (depending on how the OCR text is structured).

### **C. Thousands Indicator**
- By default, "**alreadyInThousands**" = **false**. 
- If the text explicitly states the amounts are in thousands (e.g., “in 1.000,” “tkr,” “t.kr.”), set "**alreadyInThousands**" = **true**.
- **No numeric conversion** is performed. Always return extracted values **as-is**, regardless of whether they are in thousands or not.

### **D. Missing or Unclear Data**
- If a required field is completely missing or cannot be determined from context, return an **empty string** (`""`) for that field.

### **E. Ignore Irrelevant Data**
- Disregard non-financial items, headings, metadata, or any other text not related to the fields defined above.

### **F. Output Valid JSON Only**
- Return strictly valid JSON.
- **Do not** include synonyms or extra properties — **only** the defined JSON keys plus the `"alreadyInThousands"` boolean.

---

## **3. JSON Output Format**

Your final output should be a **single JSON object** containing **all** the fields below, plus the `"alreadyInThousands"` boolean.
Replace <value> with the extracted data or "" if not found. **Numeric fields** remain numeric where possible, else as the original string if the OCR extracts symbols/decimals.
```json
{
  "companyId": "<company_id>",
  "companyName": "<company_name>",
  
  "grossProfit": <grossProfit>,
  "staffCosts": <staffCosts>,
  "otherOperatingExpenses": <otherOperatingExpenses>,
  "depreciation": <depreciation>,
  "profitBeforeInterest": <profitBeforeInterest>,
  "financialIncome": <financialIncome>,
  "financialExpenses": <financialExpenses>,
  "profitBeforeExtraordinaryItems": <profitBeforeExtraordinaryItems>,
  "extraordinaryItems": <extraordinaryItems>,
  "profitBeforeTax": <profitBeforeTax>,
  "tax": <tax>,
  "profitAfterTax": <profitAfterTax>,
  "annualResult": <annualResult>,

  "fixedAssets": <fixedAssets>,
  "currentAssets": <currentAssets>,
  "totalAssets": <totalAssets>,

  "equity": <equity>,
  "provisions": <provisions>,
  "longTermLiabilities": <longTermLiabilities>,
  "shortTermLiabilities": <shortTermLiabilities>,
  "totalLiabilities": <totalLiabilities>,
  "equityAndLiabilities": <equityAndLiabilities>,

  "alreadyInThousands": false
}
```
- Initialize `alreadyInThousands` to `false`.
- If you detect a textual marker (e.g., “t.kr.”, “tkr”, “in 1.000”) indicating the amounts are stated in thousands, set `"alreadyInThousands": true`.

No other properties or synonyms should be included in the final output.

### Quick Reference Checklist
1. **Match** fields via **exact** or **synonym** labels.
2. **Extract** numeric values exactly as found, and take the **rightmost** column if multiple appear for the same year.
3. **Do not** perform any division or rounding; always return the extracted values as-is.
4. Set `"alreadyInThousands"` to `true` if the text states amounts are in thousands; otherwise, `false`.
5. Ensure `totalAssets` aligns with `equityAndLiabilities` or `totalLiabilities`.
6. Fill in missing fields with an empty string "".
7. Output a **valid JSON** object with **no extra** properties.
