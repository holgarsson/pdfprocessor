---

## **2. Extraction Rules**


### **A. Parse & Extract**
1. In the OCR text, locate lines or labels matching **any** of the above **Field Names** or their **Synonyms**.  
2. For each matching label, extract the corresponding **numeric value**—preserving decimals, commas, and symbols exactly as seen.  
3. If the same field appears in multiple columns for the **same year**, use the **rightmost (latest) column** value.

### **B. Consistency Requirement for `totalAssets`**
- **`totalAssets`** must always match **`equityAndLiabilities`** OR **`totalLiabilities`**.  
  - This implies: `totalAssets` == `equityAndLiabilities` or `totalAssets` == `equity + totalLiabilities + provisions` (depending on how the OCR text is structured).

### **C. Thousands Normalization**
- If the amounts are explicitly stated **in thousands** (e.g., "in 1.000," "tkr," "t.kr."), **do not** convert.  
- Otherwise, **divide every extracted value by 1000**, then round to the nearest unit.  
  - **Example**: `33.014.590` → `33.015` (after rounding).  
  - **Example**: `333,123.22` → `333.123` (after rounding).

### **D. Missing or Unclear Data**
- If a required field is completely missing or cannot be determined from context, return an **empty string** (`""`) for that field.

### **E. Ignore Irrelevant Data**
- Disregard non-financial items, headings, metadata, or any other text not related to the fields defined above.

### **F. Output Valid JSON Only**
- Return strictly valid JSON.  
- **Do not** include synonyms or extraneous information in the output—**only** the defined JSON keys.

Your final output should be a single JSON object containing **all** the fields below. Replace `<value>` with the extracted data or `""` if not found.

**Note: <grossProfit>, <staffCosts>, etc. should be numeric when possible. If no value is found, output "" (an empty string).**

### Quick Reference Checklist
1. **Match** fields via **exact** or **synonym** labels.
2. **Extract** numeric values exactly as found, and take the **rightmost** column if multiple appear for the same year.
3. Apply **thousands normalization** unless the text explicitly states amounts are already in thousands.
4. Ensure `totalAssets` aligns with `equityAndLiabilities` or `totalLiabilities`.
5. Fill in missing fields with "".
6. Output a **valid JSON** object with **no extra** properties or synonyms.
